# Runtime Asset Release Sync Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ensure every WordPress release archive contains the freshly built CSS, JavaScript and image assets used by the active Logika theme.

**Architecture:** The existing artifact builder remains the single release entrypoint. It builds frontend assets, stages the three managed WordPress components in a temporary directory, overlays `build/css`, `build/js` and `build/img` onto the staged theme assets, then calculates checksums and archives that staged runtime tree. No source, uploads, database or configuration file is uploaded.

**Tech Stack:** Bash, npm/Gulp, Node.js built-in test runner, tar, WordPress theme assets.

## Global Constraints

- Release archives contain only `logika-theme`, `logika-core` and `logika-leads`.
- `wp-config.php`, uploads, database data, frontend source, tests and documentation stay out of the release.
- Build failure blocks artifact creation and deployment.
- The source worktree must not be modified by copying generated files into the theme.
- Keep existing atomic symlink deployment and manifest format compatible.

### Task 2: Build and overlay runtime assets in the artifact staging directory

**Files:**
- Modify: `scripts/release/build-artifact.sh`
- Test: `tests/release-infrastructure.test.mjs`

**Interfaces:**
- Consumes: the three existing component directories and `build/{css,js,img}` generated by `npm run backend`.
- Produces: `$staging_dir/wordpress/wp-content/themes/logika-theme/assets/{css,js,img}` before checksum creation.

- [ ] **Step 1: Stage components before checksumming**

Move the existing component copy before checksum creation:

```bash
tar -C "$source_root" -cf - "${components[@]}" | tar -C "$staging_dir" -xf -
theme_assets="$staging_dir/wordpress/wp-content/themes/logika-theme/assets"
```

- [ ] **Step 2: Build frontend output and overlay only runtime directories**

Insert after `theme_assets` is set:

```bash
(
  cd "$source_root"
  npm run backend
)

for asset_dir in css js img; do
  test -d "$source_root/build/$asset_dir"
  tar -C "$source_root/build" -cf - "$asset_dir" | tar -C "$theme_assets" -xf -
done
```

This merges generated assets without deleting theme-only runtime files such as custom section styles.

- [ ] **Step 3: Calculate manifest checksums from the staged components**

Replace the existing checksum loop with:

```bash
for component in "${components[@]}"; do
  checksum="$(tar -C "$staging_dir" -cf - "$component" | sha256sum | awk '{print $1}')"
  component_checksums+=("$checksum")
done
```

Keep the final archive command unchanged except that it now packages this fully staged tree.
